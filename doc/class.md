# Диаграмма классов (Class Diagram) — «Король Лев»

## Описание

Диаграмма классов отражает структуру системы «Иерархия наследования и управление Землями Прайда».

### Иерархия Королей
- **Абстрактный класс `King`** имеет методы `править()` и `защищатьЗемли()`.
- **Класс `Mufasa`** наследует `King`, обладает полями `сила = 100` и `мудрость = 100`.
- **Класс `Simba`** наследует `King`, обладает полями `сила = 80`, `возраст = "Взрослый"` и собственным методом `осознатьСебя()`.

### Антагонист и ресурсы
- **Класс `Scar`** не входит в иерархию `King`, но может `захватитьВласть()` и `изгнать()`.
- **Класс `PrideLands`** (Земли Прайда) — ресурс, которым управляет Король.
- **Класс `Rafiki`** — мудрый наставник, умеет `показатьИстину(с: Simba)`.

### Связи
- **Наследование:** `Simba extends Mufasa` (Симба наследует титул и обязанности).
- **Ассоциация:** `King (1) управляет (1) PrideLands`.
- **Захват:** `Scar пытается захватить PrideLands`.
- **Наставничество:** `Rafiki наставляет Simba`.

## Код PlantUML

\`\`\`plantuml

    + правило()
    + защищатьЗемли()
}

    + сила: int = 100
    + мудрость: int = 100
}

class Simba extends King {
    + сила: int = 80
    + возраст: String = "Взрослый"
    + осознатьСебя()
}

class Scar {
    + хитрость: int
    + захватитьВласть()
    + изгнать()
}

class PrideLands {
    + состояние: String
    + процветание: int
}

class Rafiki {
    + показатьИстину(с: Simba)
}

Scar --> PrideLands : пытается захватить

## Примечания

- Симба наследует роль Короля от Муфасы через механизм наследования.
- Шрам не является Королём, но может временно захватить ресурс.
- Полиморфизм: метод `править()` может работать по-разному у Муфасы и Симбы.
